# OmniFraud 接口文档（基于当前 Streamlit 代码整理）

## 1. 说明
- 当前项目是 Streamlit 单体应用，代码中未提供 Flask/FastAPI 形式的公开 REST 路由。
- 为便于前后端/队友联调，本文档按“可抽象为后端 API”的方式整理，字段严格来自现有业务逻辑。
- 文档中的接口分为：
  - 已有业务能力：代码中已存在对应函数与流程。
  - 建议封装路径：便于统一调用的 REST 风格路径（建议）。

## 2. 通用约定
### 2.1 基础信息
- Base URL（建议）：`https://<your-domain>/api/v1`
- Content-Type：`application/json`
- 字符编码：`UTF-8`

### 2.2 统一响应结构（建议）
```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

字段说明：
- `code`：业务状态码，`0` 表示成功，非 `0` 表示失败。
- `message`：状态说明。
- `data`：具体业务返回内容。

### 2.3 统一错误码（建议）
- `0`：成功
- `40001`：参数校验失败
- `40002`：输入文本过短（少于 10 字）
- `50001`：模型推理失败
- `50002`：OCR 识别失败
- `50003`：知识图谱查询失败
- `50004`：数据库连接失败
- `50005`：外部大模型服务调用失败

## 3. 短信识别模块
对应代码能力：`recognize_page.py` 中 `predict_text`、`extract_text_from_image`、`get_suggestions_stream`

### 3.1 短信文本识别
- 接口名称：短信风险识别
- 请求方式：`POST`
- 建议路径：`/sms/recognize`
- 已有能力：是

请求体：
```json
{
  "text": "【顺丰】尊敬的客户，您使用顺丰的频率较高..."
}
```

请求参数：
- `text` string 必填：待识别文本。

成功响应示例：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "prediction": "冒充电商物流客服类",
    "probability": 0.9471,
    "features": {
      "风险等级": "高风险",
      "关键词": ["支付宝", "登记", "领取"],
      "关键词风险": 88,
      "链接风险": 30,
      "紧迫性指数": 32,
      "语义异常度": 94.71
    },
    "full_predictions": [
      ["冒充电商物流客服类", 0.9471],
      ["无风险", 0.0201]
    ]
  }
}
```

返回参数说明：
- `prediction` string：预测类别（分类标签）。
- `probability` number：预测类别概率，范围 `[0,1]`。
- `features.风险等级` string：`无风险`/`低风险`/`中风险`/`高风险`。
- `features.关键词` array[string]：命中关键词列表，无命中返回 `["无"]`。
- `features.关键词风险` number：关键词风险分，范围 `[0,100]`。
- `features.链接风险` number：链接风险分，范围 `[0,100]`。
- `features.紧迫性指数` number：紧迫性分，范围 `[0,100]`。
- `features.语义异常度` number：语义异常度（模型概率*100）。
- `full_predictions` array：所有类别及概率降序结果。

失败响应示例：
```json
{
  "code": 40002,
  "message": "输入文本过短，请至少输入10个字符",
  "data": null
}
```

### 3.2 图片 OCR 文本提取
- 接口名称：截图文字提取
- 请求方式：`POST`
- 建议路径：`/sms/ocr`
- 已有能力：是

请求体（JSON 方式）：
```json
{
  "image_base64": "data:image/png;base64,......"
}
```

请求参数：
- `image_base64` string 必填：Base64 图片数据（含 mime 前缀更佳）。

成功响应示例：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "ocr_text": "您好，系统检测到您的账户异常，请立即登录..."
  }
}
```

返回参数说明：
- `ocr_text` string：识别出的完整文本。

### 3.3 识别建议生成
- 接口名称：风险建议生成
- 请求方式：`POST`
- 建议路径：`/sms/suggestions`
- 已有能力：是（流式生成）

请求体：
```json
{
  "text": "短信内容...",
  "prediction": "高风险"
}
```

请求参数：
- `text` string 必填：短信/截图合并后的文本。
- `prediction` string 必填：上游识别得到的风险等级或类别。

成功响应示例：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "suggestion": "1. 先不要点击任何链接..."
  }
}
```

## 5. 知识问答模块
对应代码能力：`bot_page.py` + `bot/agent.py` + `bot/tools/cypher.py`

### 5.1 反诈问答
- 接口名称：知识问答
- 请求方式：`POST`
- 建议路径：`/qa/chat`
- 已有能力：是（流式 + 非流式）

请求体：
```json
{
  "question": "使用手机诈骗的案例有哪些？",
  "session_id": "optional-session-id"
}
```

请求参数：
- `question` string 必填：用户问题。
- `session_id` string 选填：会话 ID；不传时可由服务端生成（当前 Streamlit 使用脚本上下文会话 ID）。

成功响应示例：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "answer": "根据知识图谱，使用手机实施诈骗的案例包括..."
  }
}
```

返回参数说明：
- `answer` string：问答结果。

失败响应示例（代码中已有兜底语义）：
```json
{
  "code": 50003,
  "message": "知识图谱查询暂时不可用",
  "data": null
}
```

## 6. 案件搜索模块
对应代码能力：`search_page.py` + `search/kg.py`

### 6.1 案件关键词搜索
- 接口名称：案件搜索
- 请求方式：`POST`
- 建议路径：`/cases/search`
- 已有能力：是

请求体：
```json
{
  "keyword": "虚假投资",
  "skip": 0,
  "limit": 30
}
```

请求参数：
- `keyword` string 必填：案件名/描述/内容关键词。
- `skip` integer 选填：分页偏移，默认 `0`。
- `limit` integer 选填：返回条数，默认 `30`。

成功响应示例：
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "total_count": 12,
    "items": [
      {
        "name": "某某诈骗案",
        "description": "...",
        "type": "刑事",
        "types": ["投资理财"],
        "subtypes": ["虚假投资"],
        "suspects": ["张三"],
        "victims": ["李四"],
        "money": 128000.0,
        "locations": ["广东"],
        "laws": ["刑法相关条款"]
      }
    ]
  }
}
```

返回参数说明：
- `total_count` integer：命中总数。
- `items` array：案件列表。
- `items[].money` number：涉案金额汇总。

### 6.2 随机推荐案件

## 9. 外部依赖接口（当前项目实际在调用）
### 9.1 OpenAI 兼容大模型接口
- 调用方式：`chat.completions.create(...)`
- 用途：
  - OCR（图片转文本）
  - 风险建议生成
  - 风险评估报告生成
  - 问答助手推理
- 关键配置：
  - `OPENAI_API_KEY`
  - `OPENAI_BASE_URL`
  - `OPENAI_MODEL`
  - `OPENAI_VL_MODEL`（可选）

### 9.2 Neo4j 图数据库
- 调用方式：Neo4j Python Driver + LangChain Neo4j
- 用途：
  - 案件搜索
  - 问答知识图谱检索
  - 图谱可视化
- 关键配置：
  - `NEO4J_URI`
  - `NEO4J_USERNAME`
  - `NEO4J_PASSWORD`
  - `NEO4J_DATABASE`

## 10. 备注（给联调同学）
- 你们如果直接访问 Streamlit Cloud 页面，不会拿到标准 REST JSON 响应，因为当前是页面驱动，不是路由驱动。
- 若需要真正前后端分离联调，建议将本文件“建议路径”在 FastAPI 中落地为真实接口，业务逻辑可直接复用现有函数。
- 建议先落地以下 4 个核心接口：
  - `POST /sms/recognize`
  - `POST /qa/chat`
  - `POST /cases/search`
  - `POST /risk-assessment/report`
这样可覆盖主要业务闭环。
